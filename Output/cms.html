Intershop Commerce Management CMS API¶

GET /cms¶

(beta) Returns the list of CMS sub resources

This operation returns a list of CMS sub resources. This includes view contexts, includes, pagelets, pages and pagetree.

Status Codes

  • 200 OK – OK

GET /cms/viewcontexts¶

(beta) Returns all view contexts of the domain

This operation returns all view contexts of the domain.

Query Parameters

  • offset (integer) – The pageable offset

  • amount (integer) – The pageable amount

  • pageable (string) – The ID of the respective pageable

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

GET /cms/viewcontexts/{viewContextID}/entrypoint¶

(beta) Resolves a page for a view context

This operation resolves a page for a view context. Depending on the type of view context query-parameters have to be specified. To resolve this call, you need all required call parameters.

Example: How to get a view context for the Catalog Cameras-Camcorder and the Category Cameras-Camcorders@inSPIRED-inTRONICS-Cameras-Camcorders

Step 1: Find out which parameters are required to resolve the view context: ``` curl /cms/viewcontexts/viewcontext.page.category.pagelet2-ViewContext —

Response:
{

… callParameter: [

{ name: “CatalogBO”, optional: false }, { name: “CategoryBO”, optional: false }, …

]

}

```

Step 2: Call this resource with at least the required parameters to receive the pagelet entry point for this view context. ``` curl /cms/viewcontexts/viewcontext.page.category.pagelet2-ViewContext/entrypoint?CatalogBO=Cameras-Camcorders&CategoryBO=Cameras-Camcorders@inSPIRED-inTRONICS-Cameras-Camcorders —

Response:

see PageletEntryPointRO

```

Parameters

  • viewContextID (string) – The view context Id. Can be qualified with the resource-set Id.

Query Parameters

  • depth (integer) – The depth of how far the slot/pagelet relations should be resolved. Further on from this depth links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 404 Not Found – Not Found

  • 400 Bad Request – Bad Request

GET /cms/viewcontexts/{viewContextID}¶

(beta) Returns a specific view context of the domain

This operation returns a specific view context of the domain. By calling a specific view context, you get a field called callParameters in the response. This is a list of objects that contains name, description and information on whether this specific parameter is optional or not.

Parameters

  • viewContextID (string) – The view context Id. Can be qualified with the resource-set Id.

Status Codes

  • 200 OK – OK

  • 404 Not Found – Not Found

  • 400 Bad Request – Bad Request

GET /cms/includes¶

(beta) Returns a list of all includes

This operation returns a list of all includes. Includes are PageletEntryPoint objects with page-flag set to false.

Query Parameters

  • offset (integer) – The pageable offset

  • amount (integer) – The pageable amount

  • pageable (string) – The ID of the respective pageable

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

GET /cms/includes/{includesKey}¶

(beta) Returns a specific include

This operation returns a specific include. The include must be published at the time of the request.

Parameters

  • includesKey (string) – The key or UUID to resolve a single item

Query Parameters

  • depth (integer) – The depth limit of how far a given include composition should be resolved. Further on from this depth links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

  • 404 Not Found – Not Found

GET /cms/pagelets¶

(beta) Returns the list of all published pagelets of the domain

This operation returns a list of all published pagelets of the domain. Pagelets which are shared to this domain will be shown as well.

Query Parameters

  • offset (integer) – The pageable offset

  • amount (integer) – The pageable amount

  • pageable (string) – The ID of the respective pageable

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

GET /cms/pagelets/{pageletsKey}¶

(beta) Returns a specific pagelet of the domain

This operation returns a specific pagelet of the domain. The pagelet must be published at the time of the request.

Parameters

  • pageletsKey (string) – The key or UUID to resolve a single item

Query Parameters

  • depth (integer) – The depth of how far the slot/pagelet relations should be resolved. Further on from this depth links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 404 Not Found – Not Found

  • 400 Bad Request – Bad Request

GET /cms/pagelets/{pageletsKey}/{slotsKey}¶

(beta) Returns a specific slot of a pagelet

This operation returns a specific slot of a pagelet. The pagelet must be published at the time of the request and the slot must exist.

Parameters

  • pageletsKey (string) – The key or UUID to resolve a single item

  • slotsKey (string) – The key or UUID to resolve a single item

Query Parameters

  • depth (integer) – The depth of how far the slot/pagelet relations should be resolved. Further on from this depth links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 404 Not Found – Not Found

  • 400 Bad Request – Bad Request

GET /cms/pages¶

(beta) Returns a list of all pages

This operation returns a list of all pages. Pages are PageletEntryPoint objects with page-flag set to true.

Query Parameters

  • offset (integer) – The pageable offset

  • amount (integer) – The pageable amount

  • pageable (string) – The ID of the respective pageable

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

GET /cms/pages/{pagesKey}¶

(beta) Returns a specific page

This operation returns a specific page. The page must be published at the time of the request.

Parameters

  • pagesKey (string) – The key or UUID to resolve a single item

Query Parameters

  • depth (integer) – The depth limit of how far a given page composition should be resolved. Further on from this depth links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

  • 404 Not Found – Not Found

GET /cms/pagetree¶

(beta) Returns all pages with their tree hierarchy

This operation returns a list of all pages that are included in the domain. Each element can have a hierarchy of optional subpages. You can limit the recursion depth by providing a depth value.

Query Parameters

  • depth (integer) – The limit of how far a given page tree should be resolved. Further on from this depth, links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 400 Bad Request – Bad Request

GET /cms/pagetree/{pagesKey}¶

(beta) Returns a specific page hierarchy

This operation returns a page found in the domain with the given identifier. It can contain a hierarchy of optional subpages. You can limit the recursion depth by providing a depth value.

Parameters

  • pagesKey (string) – The unique identifier of the requested page or a path of page identifier

Query Parameters

  • depth (integer) – The limit of how far a given page tree should be resolved. Further on from this depth, links are used instead of resource objects.

Status Codes

  • 200 OK – OK

  • 404 Not Found – Not Found

  • 400 Bad Request – Bad Request
